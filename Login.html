Use-case for login

Login

Subject: The World Wide File System

Actors involved: The user, the web browser, Application server, Database Requirements addressed:

  • As an user, I need to be able to login to my existing account or create new account.

    • Precondition: The user is not yet logged in to WWFS.

    Typical flow:

    1. The user requests the file server to loging to his account.

    2. The server opens the login page so that the user can login with his account id and password

    3. If user has already existing account he will be able to login in to it, if not he has to create new account

    4. After the user has logged in, the system provides a side bar to the user to access the files and directories for which the user has authority

    Postcondition: The user is logged in to the system
