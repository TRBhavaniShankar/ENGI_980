Use-case for Logout

Logout

Subject: The World Wide File System

Actors involved: The user, the web browser, Application server, Database Requirements addressed:

  • As an user, I need to be able to logout anytime with the choice of committing/ saving the unsaved files.

    • Precondition: The user has logged in already.

    Typical flow:

    1. The user requests the server to loggout from his account on WWFS application.

    2. If the user has some unsaved data, the system prompts the user with a message to save the data or exit

    3. If user selects "save" then the changes will be saved and simultaneously the user id logged out

    4. If the user picks "Exit", the system pops a message asking for double confirmation whether the user really wants to logg out with out saving

    5. If the user picks "Yes", the system terminates without saving the file

    6. If the user picks "No", then the system saves the file terminate

    Postcondition: "The user is logged out of the WWFS"
