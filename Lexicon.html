Lexicon for WWFS

Directory.

      It is container element, which can contain many directories or files, or it can even be empty as well.

File.

      File is a terminal element, which contains the data. It can be empty as well.

Root.

      The initial file system element(directory), which is the ultimate of all the other file system elements.

Path.

      The path which can be followed to find a directory or a file. This always starts from root.

Create Directory.

      Create new directory for the user.

Create Directory.

      Create new file for the user.

Load.

      Load the file from the directory to the user's system.

Modified.

      The user makes some changes to the existing file and the state of that file is referred to modified.

Commit/save.

      When the user creates a new file or modify the existing file it needs to be saved safely on to the WWFS database.

Delete.

      When the file or directory is not required by the user anymore, user can delete the file.

File System Manager.

      The system which controls the activities of the file system.

File transfer protocol.

      The rules and regulations used to control the flow of operation.

Eventual consistency.

      It is a consistency model used in the distributed system which helps in achieving high availability. This guarantees that if there are no new modifications committed to the file, then all the access to that file will return the updated result.
