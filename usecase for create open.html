Use-case for Create or open exisitng file/directory

Create new file/directory or open an existing file/directory

Subject: The World Wide File System

Actors involved: The user, the web browser, Application server, Database Requirements addressed:

  • As an user, I need to create new file or new directory.
  • As an user, I need to be able to open read or modify the existing file.

    • Precondition: The user is logged in to system and is able to see all the files which belongs to him/her.

    Typical flow:

    1. The WWFS system provides the user which a side bar to create a new file/directory or open an existing one.

    2. The user can create a new file by clicking on "new file" in the tool bar
          a. The system prompts the user to open the directory in where the user wants to create new file, and also asks for file details and then creates the new file.
          b. If the user does not provide the details, for example file name, the system assumes the default file details and creates the new file.

    3. The user can create a new file by clicking on "new directory" in the tool bar
          a. The system prompts the user to open the directory in where the user wants to create new directory, and also asks for file details and the creates the new directory.
          b. If the user does not provide the details then the system assumes the default details.

    4. The user can open an exisiting file by clicking on the file name (based on user needs) which is displayed in the side bar

    5. The user can open an exisiting directory by clicking on the directory name (based on user needs) which is displayed in the side bar

    Postcondition: "The user is still logged into the system"
